# Good developers should be bad at Microsoft Word

That's a strange thing to say. Why would it be good to be bad at something? Especially something that's often lauded as a skill in the digital age?

Well, its my opinion that you only have so much room in your head for so much information, as such, you should be intentional about what information you learn.

My argument is such: Microsoft Word is shite software, learning it makes you a worse professional.

## Word is bad software

I am a software developer. I trade in all the things associated with files: editing files, versioning files, generating files, fixing files, reading files. I've learned a lot of things about making good ways to store little bits of data in clean little rows that eventually turn into useful information.

Essentially, there are two types of files: binary files and text files. Along with that there is this continuum of complexity associated with the files. It goes like this:

- .txt - Text files, could be anything, could be nothing, make no expectations
- .csv - Comma Separated Value files, loosely structured files containing information
- .json - Javascript Object Notation files, a well structured file containing numerous shapes of configurations and values
- .js files - Javascript files, programming language files, these have a parsers and compilers and all the tooling to tell you if these are correct

---

At some point, these files get too complex and we decide for some reason or another that we want to turn text based files into binary files. Most people know binary means that files are all ones and zeros, but binary files are relatively simple, complex things. It just means that the files aren't text, and that whatever you use is gonna need to know whats in the file.
Here, you get things like:

- .zip - archive files, binary files where the whole of a directory is compressed into a single file, files here could be any other files
- .jpeg - Image files
- .exe - binary executables, aka runnable programs

As you go down the list, the requirements to use the file get roughly more complex. It's really simple to read in text, and csvs, but people spend a lot of time trying to find out really efficient ways to use json and js files, but whats really more complex is everything that can't just be read as some sort of text.

All the binary files require more than notepad to use, they require special little programs to read and write and make them.

I don't like requiring super special programs to read my files.

I like being able to read my files in all my programs.

## What (I) we want

- Portable - I can use it anywhere, anyhow
- Universal - If I hand it to someone, they can figure it out
- Version-able - When I make a change, I can tell that I made a change, and undo that change
